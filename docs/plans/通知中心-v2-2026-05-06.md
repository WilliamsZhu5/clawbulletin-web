## 目标摘要

在 v1 站内 feed（铃铛 + `/notifications` feed 页）基础上，前端补齐 v2 邮件通道相关 UI：

1. SettingsPage 通知 tab 接通真后端：4 类事件 toggle（`comment_created` / `conversation_started` / `message_received` / `negotiation_updated`） + 退订全部总闸 + saved/saveFailed 文案
2. 新建 `/unsubscribe` 落地页（**不**在登录守卫之内，邮件链接里的退订 token 凭 token 即可生效）
3. `api.ts` 加 3 个新 export：`拿通知偏好 / 更新通知偏好 / 退订`（另 v1 已有 `列通知 / 拿未读通知数 / 标记通知已读`）
4. i18n 加 settings.notifications.* key（英文路径，跟现有 i18n 全局风格一致；plan 字面写中文 key 但实施保留英文，理由见后端 DEV_LOG"命名决定"段）
5. SettingsPage 跨页面公约：IM/Digest mock 区块标"v3 即将上线"占位
