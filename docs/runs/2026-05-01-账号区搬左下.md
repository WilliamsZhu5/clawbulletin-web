# 账号区从 TopBar 右上角搬到 Sidebar 左下角

**日期**：2026-05-01
**触发任务**：把已登录用户的账号区从 TopBar 右上角下放到 Sidebar 左下角，符合 Slack / Discord / Notion 的业界标准。

## 改了的文件

- `src/app/components/layout/TopBar.tsx`
- `src/app/components/layout/Sidebar.tsx`

## TopBar 删了什么

- 已登录态的 `DropdownMenu`（头像 + 用户名 + ChevronDown 触发，含「个人主页 / 我的 Agent / 退出」）整块删除
- 配套的 `取首字母` / `派生头像色` / `头像调色板` 工具函数删除
- `处理退出`、`真实用户`、`显示名`、`头像首字母`、`头像色` 等局部变量删除
- 不再用的 import 清理：`拿用户`、`清登录态`、`User as UserIcon`、`LogOut`、`currentUser`、所有 DropdownMenu 系列
- **保留**：未登录态的「登录」按钮（仍在右上角，紫色 outlined）；其余 TopBar 元素（Logo、搜索框、语言切换、跟 Agent 聊、Post CTA）一律不动

## Sidebar 加了什么（已登录态）

底部块（`borderTop: 1px solid #F0F0F0`）替换为单行账号区：

- 32×32 圆形头像（紫色 `#4F46E5` 实心 + 白字首字母）
- 用户名 14px / 600 / 黑色 / 一行截断
- 副文案（email / @username / slug）12px / 400 / 灰 `#888`
- ⋯ 按钮（`MoreHorizontal`）28×28 圆形，hover `#F5F5F5`，`aria-label="账号菜单"`
- 整块 hover 浅灰 `#FAFAFA`
- padding：14px 12px

下拉菜单 `side="top" align="end" sideOffset={8}` —— 向上展开。

**同时清掉**：原 Sidebar 底部独立的「我的 Agent」「设置」两个按钮（已并入下拉菜单），避免重复。

## 头像 / 名字 / email 取值规则

按 `用户` type（src/app/data/api.ts）：

| 字段 | 优先链 |
|---|---|
| 显示名 | `display_name` → `username` → `email.split('@')[0]` → `'我'` |
| 副文案 | `email` → `@${username}` → `slug` → 空 |
| 头像首字母 | `avatar_initials` → display_name 首字符（CJK 整字 / 英文首字母大写） |
| 头像色 | `avatar_color` → `#4F46E5`（紫色 accent） |

CJK 检测正则：`/[㐀-鿿豈-﫿]/`（覆盖中日韩统一表意 + 兼容区）。

## 下拉菜单项（自上而下）

1. Header label：显示名 + email（小灰字）
2. 分隔线
3. **个人主页** → 优先 `/u/${username}`，否则 `/profile/${slug || id}`
4. **我的 Agent** → `/agents`
5. **设置** → `/settings`（用 i18n key `nav.settings`）
6. 分隔线
7. **退出**（destructive 红色样式）→ `清登录态()` + navigate(`/login`)

## 未登录态

Sidebar 左下角的「登录 / 注册」引导卡保持原样（白底 + 1px 边 + 紫色 CTA），仅外层容器 padding 微调以匹配新的 `borderTop` 节奏。

## Vite 构建

`npx vite build` ✅ 通过，2115 modules transformed，无 TS / import 错误。esbuild 单文件 syntax check 也通过。

## 已知问题

- **移动端 / 折叠 sidebar**：当前 Sidebar 是 `fixed` 220px 宽，未实现移动端抽屉；本次改动跟随原 Sidebar 容器，没有引入新的折叠状态，所以现状下移动端如果隐藏 Sidebar，账号区也跟着隐藏（与原 user chip 行为一致）。
- **`/profile/${id}` 路由兜底**：如果用户没有 username 也没有 slug，会走 `/profile/<userId>`；该路由是否存在未在本次任务核验，建议主 agent 后续 grep `path="/profile`/ 确认。
- **TopBar 右上角已登录态变疏**：按任务要求未塞 bell / 通知 icon 占位；视觉上少一个紫色 chip，整体反而更呼吸。

## 给主 agent 的"下一步"

人工浏览器验证：登录后看 TopBar 右上角无头像 chip，Sidebar 左下角出现头像 + 用户名 + email + ⋯，点 ⋯ 向上弹出 5 项菜单，点退出回 /login。
