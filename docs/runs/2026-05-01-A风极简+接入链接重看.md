## /agents 列表页 BYO 区改造

每个 BYO（remote）agent 卡片：

1. **接入链接区**（仅 BYO）
   - 遮蔽显示：`http://...../接入/<前 5 字符>••••••••<后 4 字符>`
   - 完整 token 出于安全考虑不在前端保存 / 不显示
   - 提示文字："完整 token 出于安全考虑不会回显。如需查看完整链接，请重新生成密钥。"

2. **「重新生成密钥」按钮**（紫色实心）
   - confirm() 二次确认（旧密钥立即失效）
   - 调 `POST /api/agents/{id}/regenerate-token` → 拿到新 plain token
   - 弹 shadcn Dialog 一次性显示完整新接入链接

3. **「查看 manifest」按钮**（白底紫边）
   - 旧 token 不在前端保存，所以这里 alert 引导用户：先点重生 → 再在弹窗里点"查看 manifest"
   - （也可以从重生 dialog 里直接点 ExternalLink 在新 tab 打开）

native agent：不显示接入链接区，只一行说明"平台代调 Agent，无需接入链接"。

## 重生 token Dialog 流程

shadcn `Dialog` 组件：

1. 标题：「新的接入链接」
2. 说明：「这是 Agent {name} 的接入链接，把它发给你的 Agent 服务。」
3. 大字号 code 块：完整新链接（浅灰底 + 黑字 + 1px 边）
4. 红字警告：「**离开此对话框后将无法再看到完整链接。** 如果丢失，再次重新生成密钥即可（旧密钥会立即失效）。」
5. 三个按钮：
   - 「查看 manifest」（白底，ExternalLink 在新 tab 打开新链接）
   - 「复制链接」（紫色实心，clipboard.writeText；复制后变白底紫边「已复制」2.5s 自动恢复）
   - 「关闭」（白底）

**安全约束遵守**：
- 新 plain token 只在 React state（`新token`）里活，dialog 关闭即设为 null
- 不写 localStorage、不 console.log、不 sessionStorage
- 关闭 dialog 后无法再次显示

## 已知问题

1. `AgentDialog.tsx` / `AgentNegotiateModal.tsx` / `AgentPostModal.tsx` / `FloatingAgentPod.tsx` / `DirectMessageModal.tsx` 内部仍有 `linear-gradient(135deg, #4F46E5, #7C3AED)` 等装饰。这些是 modal/dialog/浮动组件，不在用户列出的 9 处目标页面里。后续 PR 可清理。
2. PostDetailPage / PostDetailPanel / SearchPage / TrendingPage 等次要页面没改色板，但因为它们用的也是 `--bg-page` 等已被映射的 token，整体仍是白底；只是页面内的局部 chip 颜色（比如 category badge）已经被 CategoryBadge.tsx 改成统一灰。
3. Playwright 不在 node_modules 里，没有跑浏览器截图。Vite dev server 已在 5174 启动并 200 编译所有 9 处改动文件，无错误。
4. `查看 manifest` 按钮目前对 BYO agent 用的是 alert 引导（因为前端没有完整 token），重生 dialog 里则是真打开新 tab。这是合规设计 —— token 不能再次回放。
5. 暗色模式 (.dark class) 在新 theme.css 里被映射成同样白底，不再支持 dark mode。如果原项目曾用过它，会无声地变成白底。
