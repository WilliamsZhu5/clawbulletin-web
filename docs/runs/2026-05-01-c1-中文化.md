## 故意保留英文的项

1. **品牌词** — `ClawBulletin`、`talkto.me`、`OpenClaw`、`Brain OS`、`A2A`、`OAuth`、`Telegram`、`WhatsApp`、`GitHub`、`MacBook`：用户期待保留。
2. **协议/技术 token** — `API`、`URL`、`JSON`、`HTTP`、`Bearer`、`POST`：UI 文案中视为常识缩写。
3. **已废弃的 i18n key 对应英文版（translations.ts 的 en 块）** — 不动，用户切到英文仍正常。
4. **Agent 模拟对话内容（mock data）** — `MessagesPage.tsx` 中预置的 c1/c2/c3/c4 4 段英文对话历史是 mock 演示数据（非用户视角"UI 文案"，而是"假数据"），保留英文不会影响用户首次使用 UI。后续若做"Agent 自动谈判内容"用户可见的真实业务，需统一中文化。**这是本次刻意未翻的最大块。**
5. **AgentMarketplacePage.tsx** — 已基于 `zh` flag 全套中文化，未额外改动。
6. **图标内文字 `talkto.me`、`A2A`** — logo 性质，全文保留原样。
7. **shadcn/ui 基础组件、styles、routes path** — 按硬约束跳过。
8. **`Williams`、`MacBook Pro` 等示例品牌名** — 保留。

## 给主 agent 的"下一步建议"

**i18n 切换框架建议：暂不做。** 理由：
- 框架已存在（LanguageContext + useLanguage + translations）。
- 默认语言改为 zh 后，硬编码中文 + 已有 t() 调用 = 用户看到全中文。
- 真要做"中英平滑切换"，需要把所有硬编码中文都补 i18n key（约 210 个），工作量大但产品价值有限（用户已说"日常用中文电脑"）。
- 建议放到 V2/C-V2：等用户明确反馈"我需要切回英文版本来给海外朋友看"再做。

**优先建议**（按价值排序）：
1. **MessagesPage 的 mock 对话**：4 段英文对话立刻显眼，建议下一步做（30 分钟工作量）。
2. **AgentNegotiateModal 的 generateConversation()**：用户看 demo 流程时也会看到英文，建议同步处理。
3. **真 API 接入后**：用户内容（API 返回的帖子标题/正文）已是中文（postTranslationsZh），但用户的 displayName / bio / 评论是真实数据，不需要翻译。
