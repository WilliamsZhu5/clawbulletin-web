## 1. 新增统一组件清单

|---|---|

## 5. 流畅度细节

- 平滑滚动：`requestAnimationFrame` + `scrollIntoView({ behavior: 'smooth', block: 'end' })`，新增节点先布局再滚
- 乐观渲染：MyAgentChatPage 发送时立刻插入 `local-${ts}` 占位，API 回来后替换；ConversationModal 失败时把输入文本 restore
- Typing 指示器：等待 API 时显示，文案根据上下文（搜帖/起对话/思考）切换
- Hover 才显示时间戳（透明度过渡 0.18s），默认隐藏避免视觉打扰
- 连续同发送者隐藏头像和名字（更紧凑）
- ToolCallCard 用 `motion.AnimatePresence` 平滑 expand/collapse（height auto + opacity）

## 7. 已知问题 / 边界

1. **MessagesPage `agentActive` 在 mock 中固定**：界面上 agent 模式 toggle 切换不会反向影响下次回复对方头上的 agent 标记。这是 mock 数据原本的简化，未改。
2. **ConversationModal「帖子上下文 chip」**已移除（原版顶部消息流里有一个「@提及帖子」chip）。新版用 `ChatHeader` 副标题展示帖子标题，更紧凑。如果用户希望恢复那个 chip，可以加回 `MessageList` 顶部。
3. **Build 包大小警告**：vite 提示 806kB > 500kB，与本次重做无关（之前就有），可以后续 code-split 处理。
4. **Lazy 类型 import**：`MessageBubble.tsx` 和 `index.ts` 用了 `import type { 气泡角色, 气泡头像 }` 链路；TypeScript 编译没问题，build 通过。
5. **未做**：sonner toast（错误目前直接 inline 红条显示在 ChatInput 上方或当成 system 气泡）；如果要改成 toast，把 `MyAgentChatPage` 和 `ConversationModal` 的 `setErr` 替换为 `toast.error()` 即可，组件已 install。
