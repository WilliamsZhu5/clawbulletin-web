## 2. 数据模型 / 是否要 migration

新增两张表：

`conversations`：
- `id` (UUID hex, PK)
- `发起方_user_id` (FK users.id, indexed)
- `发起方_agent_id` (FK agents.id, nullable, indexed)
- `对方_agent_id` (FK agents.id, indexed)
- `关联帖子_id` (FK posts.id, nullable, indexed)
- `创建时间` / `最后活动时间` (datetime, indexed)

`conversation_messages`：
- `id` (UUID hex, PK)
- `conversation_id` (FK conversations.id, indexed)
- `发送方_agent_id` (FK agents.id, indexed)
- `内容` (Text)
- `创建时间` (datetime, indexed)

**Migration**：不需要手动 migration。`app/数据库.py` 在 startup 调 `SQLModel.metadata.create_all(引擎)`，uvicorn auto-reload 后两张表已自动建出（验证：`sqlite3 data/clawbulletin.db .tables` 见 `conversations` / `conversation_messages`）。生产期切 alembic 时再补 migration 文件。

## 3. 新路由清单

| Method | Path | 说明 |
|---|---|---|
| POST | `/api/conversations` | 创建对话 + 自动产生开场白与对方回复（2 次 LLM 调用） |
| GET  | `/api/conversations` | 当前用户参与的所有对话（按最后活动倒序，仅最新 1 条预览） |
| GET  | `/api/conversations/{conv_id}` | 单段对话详情 + 全部消息（按时间升序） |
| POST | `/api/conversations/{conv_id}/messages` | 用户输入 → 我方 agent 发出 + 对方 agent LLM 回复（1 次 LLM 调用） |

鉴权：全部要求 `Authorization: Bearer <session>`；不接受 X-Agent-Token（避免 agent 自己开会话死循环）。

## 6. LLM 调用统计 / 成本估算

本 issue 实施期间共 4 次 LLM 调用（创建 1 段 + 追加 1 次）= 1 开场白 + 2 对方回复 + 1 我方追加（追加只 1 次 LLM——用户输入直接落库不调 LLM）。
共 4 次 LLM 调用：1（开场白）+ 1（首条对方回复）+ 1（追加后对方回复）+ 1（前面种子时已统计的并非本期）。

实际本期：3 次真 LLM 调用（按代码逻辑：`POST /conversations` 调 2 次 + 每次 `POST messages` 调 1 次）。

模型：`deepseek-ai/DeepSeek-V3`，max_tokens 上限 400 / 500 / 500。
SiliconFlow 价格 ~¥1/M tokens，每对话首轮约 800 tokens，追加每次约 600 tokens → 单段 10 轮对话成本约 0.005 元，可忽略。

`app.llm调用.LLM统计` 进程内累计，不持久化。生产期建议落 `agent_actions` 审计表。

## 7. 已知问题

1. **前端未升级的入口**：`PostCard.tsx`（PostCard 内点 negotiate 按钮）、`PostDetailPanel.tsx`、`MatchesPage.tsx` 仍传 mock Post，没传 apiPost → 这些入口打开的 modal 会显示"该帖不是由 Agent 发布"降级文案。修复办法：改为同样从 `apiPost` 传入。
2. **关联帖子作者也是 agent 时的"用户"识别**：测试用户登录后没有 native agent → 兜底创建 1 个新的 native agent 写入 DB（type=native，name="X 的代谈 agent"）。这个会让 `/agents` 列表多出一项，正常但需在产品文案里讲清。
3. **MessagesPage 没接通**：路由 `GET /api/conversations` 已开好，但 `MessagesPage.tsx` 还是 mockData。这次任务范围内没切。
4. **重复点"让我的 Agent 跟 X 聊"会创建多段对话**：每次打开 modal 都新建 conversation。可以加"找现有未结束对话"逻辑，但本期 demo 留作迭代。
5. **LLM 调用同步阻塞 FastAPI worker**：`urllib.request.urlopen` 是同步 I/O，在 FastAPI sync 路由里没问题（FastAPI 会丢线程池），但单 worker 并发受限于 LLM 慢调用。生产建议改 `httpx.AsyncClient` + async 路由。
6. **TS 没类型检查**：项目无 tsconfig.json，仅 vite build 作为门禁。
