# Bulletin 前端鉴权 UX 补丁 — 2026-05-01

## 目的
- 让用户能看到「登录」入口、「退出」入口
- 401 自动跳登录、登录后跳回原页
- 不改后端、不 commit

## 改了的文件清单
1. `src/app/data/api.ts` — 401 自动跳登录 + redirect 参数校验工具
2. `src/app/components/layout/TopBar.tsx` — 登录按钮 / 用户头像下拉菜单（含「退出」）
3. `src/app/components/layout/Sidebar.tsx` — 未登录时底部显示登录提示卡
4. `src/app/pages/LoginPage.tsx` — 透传 redirect 参数到 dev_magic_link
5. `src/app/pages/MagicVerifyPage.tsx` — 验证成功后优先跳合法 redirect

## 各文件改了什么

### `api.ts`
- 在 `请求()` wrapper 里：fetch 后检查 `resp.status === 401`，若是：
  - 调 `清登录态()` 清掉 localStorage 中的 token + 用户
  - 用 `window.location.assign('/login?redirect=<encodeURIComponent(当前路径)>')` 同步硬跳
  - 防循环：当前已经在 `/login*` 路径时不跳，照常抛 401 给 LoginPage 自己处理
- 新增并 export `是否合法redirect(p)`：只允许以单个 `/` 开头、排除 `//` 与 `/\` 的字符串（防 open redirect 漏洞）

### `TopBar.tsx`
- 新增 import：`已登录 / 拿用户 / 清登录态`、shadcn `dropdown-menu`、`LogIn / LogOut / UserIcon` 图标
- 右上角根据登录状态条件渲染：
  - 未登录 → 「登录」按钮（白底 + 边框，点 navigate('/login')）
  - 已登录 → 圆形头像（真实 `avatar_initials` + `avatar_color`）+ display_name + ChevronDown，点开 DropdownMenu 含：
    - 标题（display_name + email）
    - 「个人主页」（仅 username 存在时）
    - 「我的 Agent」
    - 「退出」（destructive variant，点击 `清登录态()` + navigate('/login')，无确认弹窗）
- 保留了原有 mockData `currentUser` import（作为 avatar_color 兜底）但实际已经用真实用户数据
- 保留了 Avatar 旧的"点头像跳 /u/..."逻辑被替换为下拉菜单触发

### `Sidebar.tsx`
- 新增 import：`已登录 / 拿用户`、`LogIn` 图标
- 底部 user chip 改为 IIFE 条件渲染：
  - 未登录 → 渐变紫色提示卡：「登录后可以创建你的 Agent，让它替你发帖、搜索、联系其他 Agent」+ 渐变「登录 / 注册」按钮
  - 已登录 → 显示真实用户头像 + display_name/username + email（兜底 currentUser.talktoLink）

### `LoginPage.tsx`
- 新增 import：`useSearchParams`、`是否合法redirect`
- 读 `?redirect=` 参数，校验后存为 `redirect目标`
- 新增 `拼上redirect()` 工具函数：把 redirect 参数附加到 dev_magic_link 上
- `handleEmailSubmit` 拿到 `dev_magic_link` 后用 `拼上redirect` 拼接，再 setState；这样用户点击 dev magic link 跳到 /login/verify 时也带着 redirect

### `MagicVerifyPage.tsx`
- 新增 import：`是否合法redirect`
- 验证成功后跳转优先级改为：
  1. URL 上有合法 redirect 参数 → 跳 redirect（站内）
  2. 否则按原逻辑（无主动建过 agent → /agents/new；有 → /）
- 首次创建 agent 引导逻辑完整保留

## 验证

- 用 Vite (esbuild) 做前端 transpile（项目无 standalone tsc，无 tsconfig.json）
- 通过 `curl http://localhost:5173/<改动文件>` 拉取 transformed module，5 个文件均 HTTP 200，body 是合法 JS module，无 Vite Transform failure / SyntaxError
- 未在主 agent 中执行浏览器 E2E（需要人工或 Playwright，本任务范围外）

## 已知问题

1. 用户登出后 TopBar/Sidebar 立即重渲染依赖 navigate 触发 react-router 状态更新；如果某个组件未被路由变化重渲染（罕见），可能短暂显示已登录状态。dev UX 可接受。
2. `已登录()` / `拿用户()` 每次渲染都读 localStorage（同步、便宜），未做 React Context；多 tab 同步未考虑，dev 环境足够。
3. 401 redirect 用 `window.location.assign` 而非 react-router `navigate`，会触发整页刷新、清空 in-memory state；优点是简单可靠不会因组件卸载导致 race；缺点是稍微重一点。任务说明中「保持简单优先就直接跳」，已选此方案。
4. talkto.me 验证那一支线 `handleTalktoConfirm` 仍然 `navigate('/')`，没有处理 redirect 参数。该支线目前是 mock，不影响真实流程。
5. 登录入口的「不登录直接浏览 Bulletin」按钮 navigate('/')，未透传 redirect — 这是用户主动放弃登录，符合预期。

## 给主 agent 的下一步建议

- 真要做浏览器 E2E：参考 `~/.claude-config/rules/ui-e2e-testing.md`，用 Playwright + Claude vision 跑七步验收（登录入口可见、点登录跳 /login、magic link 验证成功跳合理页、登录后头像下拉、退出回 /login、401 模拟跳 /login?redirect=...、登录后回原页）
- 如果后端要把 `magic-link` API 中 `dev_magic_link` 改成"由后端预先拼好 redirect"，前端 `拼上redirect` 这层可以删除，但目前接口契约是后端不知 redirect，前端拼接是正确分工
- 全站还有一些 mockData `currentUser` 残留（`PostCard / ProfilePage / SettingsPage` 等），未在本次改动范围内；后续可以做一轮"切换到真实用户"的统一迁移
- 多 tab 登录态同步可加一个 `storage` event listener 推动重渲染（成本低、收益小，按需）
