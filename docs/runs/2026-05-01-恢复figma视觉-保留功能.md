## 已知问题（**不要自动 fallback 到 mock，等主 agent 决定**）

1. **ConversationModal 仍是 figma 的 fake-LLM 演示**。figma 提交的版本里这个组件用 `theirReplies[]` / `agentAutoReplies[]` 数组生成假对话，没有调 `发起对话` / `追加消息` API。要恢复 A-Conv 的真 LLM，需要把这两个调用织入到 `setMessages` 的位置（约 600 行的组件）。建议主 agent 派一个专项 sub-agent 来做这个 patch，不要让它在视觉恢复任务里偷做。

2. **AgentDialog / AgentNegotiateModal / AgentPostModal / DirectMessageModal / FloatingAgentPod** 也都是 figma 的 fake 演示（这些 modal 在 figma 提交里就是脚本式假对话）。如果"5-agent 自主活动"功能依赖这些 modal 的真 API 调用，需要单独 patch；如果只依赖后端定时任务则不影响。

3. **AgentSetupPage / List / Done / MyAgentChat / 复制按钮** 视觉只做了 token 级替换，间距 / 阴影 / 字体没有逐行对齐 figma design system。如果需要 pixel-perfect figma 风格，建议派 UI sub-agent 重写。功能层完全保留，可用。

4. **CreatePostPage 的「让 Agent 起草」入口** 是用 figma 调色板加的紫色渐变卡片，但与 figma 原版的 "Choose a category" 表单同处一屏。视觉融合度尚可，但不是 figma 原稿里的元素。如果 figma 设计师后续给到正式起草页设计稿，可替换。

5. **mockData.ts 里的 `currentUser`** 仍被 PostDetailPanel / DirectMessageModal / ProfilePage / FloatingAgentPod / ConversationModal 当成"当前登录用户"使用。这些组件在已登录态下显示的不是真实用户而是 mock 名字。Sidebar 的账号区已用 `拿用户()` 真实数据，但其他组件还没改。建议主 agent 安排一个清理任务把所有 `currentUser` 引用替换为 `拿用户()`。
