## 3. 没改的 mockup（明确说明原因）

- **侧栏「Williams / talkto.me/williams」用户卡**（layout）— 仍用 `currentUser` 占位；登录态接入应在 layout 组件做，超出 M1b "首页/详情/列表"重点范围
- **Trending 页右侧「Trending Tags」**仍用 `mockData.trendingTags` 的硬编码数组；后端没有 trending-tags endpoint
- **Trending 页「Active listings 147 / New discussions 89」三个统计卡片**是写死的展示数字；后端没有 stats endpoint
- **TrendingPage 的热度算法**目前用 `sort=hot` 后端排序后客户端再 score，可改进但不阻塞
- **MessagesPage / MatchesPage / ProfilePage / MyPostsPage / SavedPage**：本轮范围明确不含，仍用 mockData
- **DirectMessageModal / ConversationModal / FloatingAgentPod / AgentDialog 等组件**：仍用 `currentUser` 占位头像和写死 posts 数组。这些是 modal/聊天 UI，不在 M1b 路径上
- **agent 列表 UI**：`GET /api/agents` 后端要求 `Authorization`（401 without token），登录后才能展示。前端尚未有 "我的 agents" 页面入口；需 M2 加专门页面

## 4. 关键发现 / M2 阻碍候选

- **后端 `GET /api/agents` 强制 auth**（401 缺 Authorization）。如果产品要求"未登录用户也能浏览有 agent 作者的帖子"，那当前帖子 API 已经在 `author` 字段嵌入 agent 名（如 "舆情风" / "码界"）已经满足；如要单独的 agent 列表/详情页，必须先登录
- **后端无 CORS 配置**。前端浏览器请求 `http://127.0.0.1:8001` 会被预检拒绝。本轮用 Vite dev-proxy 绕过（`/api` → `127.0.0.1:8001`），生产部署需要后端加 `CORSMiddleware` 或前端 nginx reverse proxy
- **`/api/auth/magic-link` 返回的 `dev_magic_link` 写死 `localhost:3000`**（应是后端配置值）。当前应该指向 `localhost:5173`。前端用户复制链接后需手动改 host 才能登录。建议 M2 在后端把 dev URL 配置走 env，或前端 LoginPage 拦截显示时替换 host
- **Talktome agent 的 `/api/comments?post_id=...` 这种 query 形式不存在**：后端实际是 `/api/posts/{post_id}/comments`。前端已用对的形式（`列评论` 函数），任务描述里的 query 写法是 audit 偏差
- **`mockData.ts` 仍存在且导出 currentUser/posts/authors**：保留是因为 PostCard 等组件依赖 `Post` 类型，且 currentUser 还在多处占位。彻底删除是 M2 工作

**登录流程**：

3. 页面会显示"⚠️ DEV 模式"黄框 + 一个 magic link

或者更快：在 terminal 里 `curl -X POST http://127.0.0.1:8001/api/auth/magic-link -H 'Content-Type: application/json' -d '{"email":"YOU@example.com"}'`，从返回 JSON 复制 `dev_magic_link`，把 `:3000` 改成 `:5173` 后浏览器打开。
